# 文脈 BUMMYAKU - 多语言文献翻译识别智能助手

**文脈 (BUMMYAKU)** 是一款专门为学术研究、历史文献以及文学作品打造的高精度翻译系统。它针对复杂的多语言排版（如 PDF/图像）进行了专门优化，使用**“高精度视觉 OCR 解析 + 大模型上下文关联翻译”**的双轨一键执行流水线，并支持智能级联的批量翻译与双语对照导出。

---

## 🌟 核心亮点与特色

- **双轨一键执行 (Dual-Workflow)**：一键完成当前页面“视觉识别 (OCR) ➔ 高精大模型翻译”全套工序，并将识别和翻译成果精准存留。
- **上下文级联翻译**：在批量翻译学术文献时，后续页面将自动引入前文的“专业历史名词、术语表、核心句意指涉”等作为级联微调背景，彻底消灭上下文脱节现象。
- **极简化学术质感 UI**：采用温暖护眼的经典学术纸张底色与优雅的衬线字体排版，界面布局经过极致压缩与对称美化：
  - **对称极简工具栏**：左侧 5 个核心资产管理功能（导入、源/译语言选择、文献信息、清除缓存），右侧 5 个核心执行与出块按键（页码、双步一键、范围设定、批量级联、本页/全局导出），美观与高效并存。
  - **宽级联范围设定**：为页码范围输入框预留了宽裕的可读空间，有效提升了大页码数的输入体验。
- **多维度成果导出**：支持当前页面或整篇文献一键生成并导出 **“纯原文”**、**“纯译文”** 以及 **“双语对照”** 格式，完美满足论文引用或译作排版需求。

## 🛠️ 简易分步操作说明

### 第一步：导入文献
点击左侧最瞩目的 **“导入文献 (PDF/图片)”** 按钮，上传需要翻译的学术文献（支持 PDF 或图片文件）。

### 第二步：配置语言与模型
1. **选择语种**：在工具栏的 **源** (Source) 和 **译** (Target) 下拉菜单中，设定文献当前的原语言与您期望翻译的目标语言。
2. **选择模型**：点击右上角的 **“模型配置”** 按钮。
   - **OCR 引擎**：选择高精度视觉解析模型（例如：*Gemini 2.5 Flash* ）。
   - **翻译引擎**：选择深度学术文献翻译引擎（例如：*Gemini 2.5 Pro* 等深度推理模型）。
   - 设置完成点击保存。

### 第三步：页面交互与一键执行
* **单页逐个突破**：
  1. 使用页码控制按键切换到指定页面。
  2. 点击右侧工具栏的 **“双步一键执行”** 按钮，系统将自动进行：
     - *左侧：视觉识别 (OCR)* 提取原文。
     - *右侧：优雅排版翻译* 产出高精译文。
  3. 您可以直接在提取到原文或翻译框中进行**手动编辑与精校**，并一键复制内容到剪贴板。

### 第四步：批量自动化级联翻译
* 如果文献有多页且需要保持上下文术语统一：
  1. 在 **“范围”** 输入框中，输入需要翻译的起始页码与截止页码（输入框已调宽，支持长页码）。
  2. 点击 **“批量级联翻译”**，系统将启动自动化不间断流水线。后一页的翻译会自动参考前一页已经生成的翻译上下文，确保专业术语在整篇文献中高度一致。

### 第五步：多格式成果导出
* 点击右侧的 **“导出本页”** 或 **“导出全部”**：
  - **仅原文**：导出纯文本原始文献内容。
  - **仅译文**：导出干净整洁的完整中/英文学术译稿。
  - **双语对照**：生成完美的段落对照版（每段原文下方紧跟对应译文），非常适合精读与对照研究。

## 📦 项目本地开发与运行

如果您想在本地克隆并运行本项目，可以使用以下命令：

```bash
# 1. 安装项目依赖
npm install

# 2. 启动本地开发服务 (支持前端预览与 API 反代服务器)
npm run dev

# 3. 编译打包项目 (打包前端 SPA 与后端服务端)
npm run build

# 4. 生产环境启动
npm start
```

*注：项目服务默认绑定在 `3000` 端口上。*
